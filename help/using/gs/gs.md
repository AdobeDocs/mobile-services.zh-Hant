# 入門{#getting-started}

此資訊可協助您瞭解及使用 Adobe Mobile Services。

Adobe Mobile Services 包含以下項目:

* Adobe Mobile Services 使用者介面
* Adobe Mobile SDK

如果企業公司想要以最有效率的方式增加使用者互動，並證實行動應用程式的投資報酬率 (ROI)，Adobe 提供的 Adobe Mobile Services 就是理想的端對端解決方案，不僅能贏取應用程式使用者並吸引他們參與，還可以分析及最佳化使用者體驗。

現今的行動應用程式市場已與當初智慧型手機剛推出時的市場大不相同。現在只透過行動應用程式保持客戶與品牌之間的連結是不夠的；您必須確保每個管道都能提供一致且令人信服的客戶體驗，並利用行動應用程式做為策略接觸點，吸引最忠誠的高價值客戶參與。而如果想吸引這些使用者與您的應用程式互動，就需要有精彩的內容，以及上下文通知、智慧個人化和整合式應用程式分析等功能。

## Adobe Mobile Services 使用者介面 {#mobile-services-ui}

以下瀏覽器可支援 Mobile Services 使用者介面:

* Google Chrome (最新的兩個版本)
* Mozilla Firefox (最新的兩個版本)
* Apple Safari (最新的兩個版本)
* Microsoft Edge (最新的兩個版本)

Adobe Mobile Services 可透過以下方式提高行動應用程式的互動率:

### 贏取

在&#x200B;*「贏取」*&#x200B;階段，您可使用已經付費、擁有和賺取的媒體，在各個主要應用程式商店中贏取使用者下載應用程式。Adobe Mobile Services 能促進應用程式使用者的贏取程序。

Adobe Mobile Services 提供使用者贏取工作流程，其中包括贏取追蹤和深層連結，可讓您衡量應用程式使用者贏取管道的效益。透過追蹤哪些使用者來自哪個通道的行銷連結，您可以洞悉哪些贏取通道對推動獲利及參與度最高的使用者。

此外，深層連結則可將使用者直接導向您想讓他們看到的應用程式內容，且您可視需要鼓勵他們安裝應用程式。

「贏取」階段提供下列主要功能:

* 應用程式贏取分析
* 各應用程式商店的追蹤連結
* 應用程式深層連結
* 與廣告網路的回傳整合

如需此階段的詳細資訊，請參閱 [贏取](/help/using/acquisition-main/acquisition-main.md).

### 分析

在&#x200B;*「分析」*&#x200B;階段，您可以瞭解消費者使用行動應用程式的方式，以及促使他們轉換或再度造訪的原因。

透過 Adobe Analytics，您可以深入洞察漏斗中的關鍵分析資料，瞭解使用者如何下載、安裝和開啟應用程式。此外，您還可衡量並分析應用程式內容及使用者介面，以及執行群眾、路徑和流失分析。Adobe Analytics 可讓您使用集中式資料存放區發佈行銷決策通知，並減少組織中的行銷資料倉儲數量。

您可使用 Adobe Audience Manager 讓對象區段充滿有意義的資料 (rich data)，進而提供背景內容更豐富的個人體驗。

*「分析」*&#x200B;階段提供下列主要功能:

* 應用程式互動分析
* 路徑和漏斗分析
* 群眾和保留分析
* 位置分析
* 支援多種裝置和平台

如需有關可執行並用來分析客戶的報表詳細資訊，請參閱 [報表](/help/using/usage/usage.md).

### 參與

在&#x200B;*「參與」*&#x200B;階段，您可使用相關的推送通知和應用程式內訊息與使用者通訊。善用目標推送通知和應用程式內傳訊功能，可確保使用者持續返回使用您的應用程式。藉由 Analytics 的區段支援，您可以指定將推送通知傳送給會回應的使用者區段，進而提高他們轉換的機率。

*「參與」*&#x200B;階段提供下列主要功能:

* 分析區段可觸發推送通知。
* 即時分析、警告和新項目/內容可觸發應用程式內訊息。
* 瞭解檢視、點進率和下游行為

### Adobe Mobile訊息

您可使用推送通知和應用程式內訊息與使用者通訊。推送通知會透過裝置的作業系統傳送，而應用程式內訊息則會在使用者與應用程式互動時，直接在應用程式內傳送。應用程式內訊息通常可透過多種其他格式呈現，包括彈出式視窗和插入式連結。

在 Adobe Mobile 中，您可以設定下列類型的訊息:

**推送通知**&#x200B;會在應用程式外部顯示，並提供下列功能:

* 透過相關的推送通知促進重複參與率。
* 建立和區隔訊息，並將訊息傳送給已下載某品牌應用程式，且已透過選擇加入同意接收推送通知的客戶。
* 由應用程式商店從伺服器端傳送，而非從行動應用程式傳送。

如需如何建立推送通知的詳細資訊，請參閱[建立推送訊息](/help/using/in-app-messaging/t-create-push-message/t-create-push-message.md)。

**應用程式內通知**&#x200B;提供下列功能:

* 使用者在應用程式工作階段時，吸引他們進行特定動作。
* 由於訊息是由應用程式 (而非推送傳遞網路) 傳送，因此可使用其他格式如警告或全螢幕。
* 即時分析可觸發應用程式內通知。
* 允許交叉促銷應用程式和產品。
* 鼓勵使用者留下應用程式商店評分。
* 根據位置提供即時訊息

如需如何建立應用程式內訊息的詳細資訊，請參閱[建立應用程式內訊息](/help/using/in-app-messaging/t-in-app-message/t-in-app-message.md)。

### 最佳化

在&#x200B;*「最佳化」*&#x200B;階段，您可以最佳化轉換 (訂閱、商務交易和收入等等) 並提升客戶保留率。最佳化使用者在您應用程式內的體驗，有助於您提供個人化內容，進而創造最高的投資報酬率和轉換率。

如需有關測試和 Adobe Target 的詳細資訊，請前往 [Adobe Target](https://docs.adobe.com/content/help/en/target/using/target-home.html)。

### 地理定位

透過 GPS 定位技術，行動裝置可讓您直接掌握消費者與您的應用程式互動時，或在背景中執行您的應用程式時，他們的所在位置。Adobe Target 利用地理定位技術，讓您在使用者進入鄰近範圍時，提供量身打造的相關內容、優惠或訊息。您可以指定位於定義的地標半徑內，或進入 iBeacon 鄰近範圍並收到相關推送通知時的使用者。

適用於行動應用程式的 Adobe Target 現可充分運用 Adobe Analytics 提供的區隔及報告增強功能。這表示 Adobe Target 可以利用 Analytics 中的所有關鍵應用程式量度進行目標及個人化設定；也可以在測試成功時提供更深入的分析報告，讓行銷人員能更有效瞭解各種「假設」問題，這些問題的解答將能減輕應用程式行銷人員的壓力，也有助於促進應用程式的投資報酬率。適用於應用程式整合的 Analytics/Target 兩者是可搭配運用的產品，也是目前市場上最健全的應用程式參與解決方案。

* Android SDK 指南中的[位置一節](/help/android/location/location.md)

## Adobe Mobile SDK {#mobile-services-sdk}

Adobe 提供端對端行動行銷解決方案，可提升客戶在所有行動領域的參與率。透過一個 SDK，您就可以存取 Adobe Analytics、Adobe Campaign 和 Adobe Audience Manager 的功能，能減少管理多個不同 SDK 的技術成本。

Adobe Mobile SDK 提供下列功能:

* 端對端行動參與

   您可使用一個整合式輕量 Adobe Mobile SDK 衡量並最佳化不同平台的應用程式。
* 贏取新客戶並製造吸引客戶參與的微時刻 (micro moment)

   * 透過目標式推送通知 (包括多媒體支援和應用程式內傳訊) 吸引使用者重複返回使用您的應用程式。
   * 利用深層連結，將應用程式使用者直接導向您想要他們看到的內容。

* 衡量並最佳化體驗，進而提高投資報酬率

   您可以深入洞察應用程式生命週期量度，包括漏斗分析 (下載、安裝、開啟)、動作分析 (工作階段長度、當機、信標) 及傳訊互動資料。
* 全方位支援

   * 支援多種主要行動作業系統和跨平台開發工具。
   * 支援智慧型手機、平板電腦、穿戴式裝置和 OTT (over-the-top) 主控台等多種裝置。

* 經過整合

   * 一個 SDK 適用多個解決方案 (Analytics、Campaign 和 Audience Manager)，可減少開發人員的實施時間和人力。
   * 只要一行程式碼就可以收集「基線」應用程式生命週期量度。
   * 行動策略改變時，您可以輕鬆啟用 Adobe Experience Cloud 功能來贏取和分析使用者，並吸引使用者參與。

* 快速且輕量

   * 將傳送資料至 Adobe 伺服器及第三方系統時的裝置處理負載量降到最低。
   * 佔用的記憶體空間很少，因此可將提交到應用程式商店的應用程式套件大小降到最小。

如需 Adobe Mobile SDK 的詳細資訊，請參閱[適用於 Experience 解決方案的 Android SDK 4.x](https://docs.adobe.com/content/help/en/mobile-services/android/overview.html) 和[適用於 Experience Cloud 解決方案的 iOS SDK 4.x](https://docs.adobe.com/content/help/en/mobile-services/ios/rel-notes.html)。
